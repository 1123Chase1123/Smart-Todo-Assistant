# Smart Todo Assistant
一款基于 **HarmonyOS Next (API 13)** 开发的轻量化离线待办事项管理应用，使用 ArkTS 语言构建，专为鸿蒙设备设计。

---

## ✨ 项目简介
本项目是一款简洁高效的待办事务管理工具，面向学习、工作与日常生活场景，提供任务全生命周期管理、数据本地持久化与完成进度可视化统计功能。应用全程离线可用，界面贴合鸿蒙原生交互规范，运行稳定、资源占用低，帮助用户高效规划时间、提升事务管理效率。

---

## 🚀 核心功能
- **完整任务管理**：支持任务的新增、查看、编辑、删除与完成状态切换，可自定义任务名称、优先级与创建时间。
- **智能筛选排序**：按任务完成状态、优先级快速筛选，支持自定义排序规则，高效梳理待办事项。
- **数据持久化存储**：基于鸿蒙 Preferences 接口实现数据本地存储，应用重启或设备重启后数据不丢失。
- **可视化统计面板**：直观展示总任务数、已完成/未完成任务占比，清晰掌握事务完成进度。
- **鸿蒙原生适配**：遵循 ArkTS 强类型语法规范开发，适配系统组件调用规则，交互逻辑贴合原生使用习惯。

---

## 🛠️ 技术架构
- **开发环境**：DevEco Studio, HarmonyOS Next API 13, ArkTS
- **组件体系**：采用稳定成熟的 `@Component` 传统组件架构，解决跨组件回调传递与状态联动问题。
- **状态管理**：使用 `@State`/`@Prop`/`@ObjectLink` 实现响应式状态管理，精准驱动 UI 刷新。
- **数据规范**：自定义数据实体接口，摒弃 `any` 类型，严格遵循 ArkTS 强类型检查规则。
- **数据存储**：封装 Preferences 工具类，统一完成任务数据的序列化、存取与解析操作。
- **问题优化**：解决 `this` 上下文绑定错乱、组件属性调用错误等编译与运行异常，保障应用稳定运行。

## 📁 项目结构
```
Smart-Todo-Assistant/
├── model/              # 数据模型层
│   └── Task.ets        # 任务实体类定义
├── utils/              # 工具类层
│   └── Database.ets    # 数据持久化工具
├── components/         # 自定义组件层
│   ├── TaskItem.ets    # 任务列表项组件
│   └── StatisticsPanel.ets # 数据统计面板组件
└── pages/              # 业务页面层
    └── Index.ets       # 主页面（业务逻辑入口）
```

## 📦 快速上手
1.  **环境准备**：安装 DevEco Studio，配置 HarmonyOS Next API 13 开发环境。
2.  **导入项目**：将项目代码克隆至本地，在 DevEco Studio 中打开项目文件夹。
3.  **构建运行**：连接鸿蒙设备或启动模拟器，编译并运行应用。
4.  **开始使用**：创建、编辑待办任务，体验筛选、排序与数据统计功能。
